# Reportes semanales SEMSE

Carpeta para llevar control semanal del trabajo realizado, lo pendiente y el plan de la siguiente semana.

## Ritmo recomendado

- Cierre de reporte: lunes al final del dia.
- Periodo cubierto: martes a lunes, o el bloque semanal que se defina antes de empezar.
- Responsable sugerido: agente de control semanal.
- Fuente principal: commits, PRs, issues, estado de Git, CI/deploy y notas operativas.

## Archivos

- `2026-06-09_a_2026-06-22_reporte-bisemanal.md`: reporte inicial de las ultimas dos semanas.
- `2026-06-23_estado-pendientes.md`: estado actualizado de pendientes despues de revisar PR, CI y deploy.
- `2026-06-23_reconciliacion-backlog.md`: plan para actualizar el backlog canonico desfasado.
- `plantilla-reporte-semanal.md`: formato reutilizable para cada semana.
- `plan-agente-control-semanal.md`: reglas para que un agente mantenga el control y prepare notificaciones.
- `configuracion-notificaciones.md`: correo destino, cadencia y regla de envio.

## Flujo semanal

1. Revisar actividad de Git y GitHub.
2. Separar entregado, en progreso, bloqueado y pendiente.
3. Registrar riesgos y decisiones.
4. Definir plan de la semana siguiente con responsables.
5. Preparar resumen corto de notificacion para enviar al usuario.
