# Execution History

Histórico de ejecución del programa.

Esta carpeta ya no debe leerse como trabajo activo. Conserva trazabilidad de planes y backlogs previos.

## Subáreas

- `backlogs/`
  backlog históricos por sprint o frente.
- `plans/`
  planes históricos de integración, hardening o ejecución.

## Regla

Si un documento gobierna el trabajo actual, no debe vivir aquí. Debe promocionarse a la raíz de `execution/` o al lugar activo correspondiente.
