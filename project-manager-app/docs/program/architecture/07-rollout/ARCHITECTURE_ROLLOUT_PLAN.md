# Architecture Rollout Plan

## Goal

Complete the new architecture of the construction operating system first, then attach it cleanly to the broader ecosystem.

## Phase A — Formalize architecture
- groups
- roles
- toolkits
- modules
- permissions
- mappings

## Phase B — Map current product
- classify pages and flows in `labsemse/src`
- define which toolkit each page belongs to
- identify missing modules vs existing pages

## Phase C — Align navigation
- role-aware dashboard structure
- module-based menus
- visibility by toolkit and role

## Phase D — Align backend/RBAC
- map permission matrix to guards and policies
- define ownership boundaries
- define finance/QA/dispute separations

## Phase E — Ecosystem attachment
- connect the formal construction OS architecture back into the wider SEMSE ecosystem
- preserve ecosystem vision while using the architecture as operational precision layer

## Rule of rollout

First complete the internal architecture.
Then attach it to the ecosystem.
Not the other way around.
