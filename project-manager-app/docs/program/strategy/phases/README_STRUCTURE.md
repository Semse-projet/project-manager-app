# LABSEMSE Program Structure

## Purpose

This folder holds the structured planning, governance, execution, and status artifacts for SEMSE.

## Sections

### `strategy/`
High-level framing and consolidation documents.

Contains:
- `SEMSE_CONSOLIDATION_PLAN.md`
- `SEMSE_EXTRACTION_MATRIX.md`

### `execution/`
Operational plans, sprint plans, execution boards, and implementation backlog.

Contains:
- `SEMSE_INTEGRATION_BACKLOG.md`
- `SEMSE_SPRINT_1_PLAN.md`
- `SEMSE_SPRINT_1_TICKETS.md`
- `SEMSE_FASTLANE_PLAN.md`
- `SPRINT_1_5_EXECUTION_BOARD.md`

### `governance/`
Role, permission, and control models for SEMSE.

Contains:
- `SEMSE_PERMISSION_MATRIX.md`

### `status/`
Rolling status and current-state documents.

Contains:
- `SEMSE_SPRINT_STATUS.md`

## Rule

Program-level documents that become part of SEMSE source-of-truth should live here, not only in the OpenClaw workspace.

The OpenClaw workspace remains a staging and operator area; `labsemse/program/` is the project-facing location.
