# SEMSE Demo Script — 5 Minutes

## Apertura (20-30s)

SEMSE no es solo un marketplace para publicar trabajos.
SEMSE conecta el ciclo operativo completo: intake comercial, ejecución por milestones, evidencia verificable, control financiero por escrow y manejo visible de excepciones.

Lo que voy a mostrar ahora es una demo reproducible del flujo principal.

---

## 1. Overview ejecutivo (30-45s)

Abrir la home.

Narrativa:
- Aquí se ve la capa comercial mínima del MVP.
- La lógica central está organizada alrededor de jobs reales, no de abstracciones vagas.
- La demo viene sembrada con dos casos: uno de happy path y otro de excepción.

Mensaje clave:
- **SEMSE opera el trabajo, no solo lo lista.**

---

## 2. Happy path job-first (60-75s)

Entrar a `Kitchen Remodel — Orlando`.

Narrativa:
- Este job representa el camino canónico.
- Desde aquí se ve el trabajo, los milestones y las acciones de revisión.

Crear milestone:
- title: `QA final walkthrough`
- amount: `900`
- sequence: `3`

Luego ejecutar:
- `Submit`
- `Approve`
- `Release`

Mensaje clave:
- **El milestone no es solo una tarea; es una unidad operativa y financiera.**

---

## 3. Evidence (35-45s)

Ir a `Evidence`.

Registrar una evidencia:
- kind: `DOCUMENT`
- filename: `demo-closeout.pdf`

Narrativa:
- Cada entrega importante puede dejar evidencia trazable.
- Esto mejora aprobación, auditoría y reduce fricción en pagos.

Mensaje clave:
- **SEMSE vincula ejecución con prueba verificable.**

---

## 4. Escrow (35-45s)

Ir a `Escrow`.

Fondear:
- amount: `1500`

Narrativa:
- El dinero sigue reglas del job y del milestone.
- El release se entiende dentro de un circuito controlado, no como un pago suelto.

Mensaje clave:
- **SEMSE ordena confianza operativa y financiera en la misma superficie.**

---

## 5. Dispute / excepción (45-60s)

Volver y abrir `Roof Repair — Miami`.

Narrativa:
- No basta con mostrar un sistema cuando todo sale bien.
- También hay que operar excepciones de forma visible.

Mostrar disputa abierta y resolverla.

Mensaje clave:
- **SEMSE también administra el conflicto, no solo el happy path.**

---

## Cierre (20-30s)

Resumen:
- job-first execution;
- milestone review;
- evidence;
- escrow;
- dispute handling.

Cierre sugerido:
Esta build no pretende enseñar todo el futuro de SEMSE.
Pretende demostrar que el núcleo del flujo principal ya puede mostrarse de forma clara, reproducible y presentable.
