# Planning

Esta carpeta agrupa artefactos previos a ejecución:

- backlogs;
- blueprints;
- mapeos;
- DTO packs;
- historias;
- planes tácticos.

Si el trabajo ya fue ejecutado, el cierre correspondiente debe vivir en la raíz de `reportes/`.

## Qué no debe confundirse aquí

- `planning/` no es canon estructural;
- `planning/` no es evidencia de trabajo terminado;
- `planning/` no reemplaza a `program/`.

## Relación correcta

- diseño estable y dirección → `program/`, `vision/`, `constitution/`, `agents/`
- evidencia de lo ejecutado → `reportes/`
- insumo previo al trabajo → `reportes/planning/`
