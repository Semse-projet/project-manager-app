# Reporte — Remediación 1.13: Nombre interno del algoritmo expuesto en UI

**Fecha:** 2026-07-23  
**Repositorio:** Semse-projet/project-manager-app  
**Ítem:** `docs/AUDIT_REMEDIATION_PLAN.md` — Sección 1, ítem 1.13 (MEDIO)  
**Autor:** Devin

## Problema

La pantalla `/client/professionals` exponía el nombre técnico del algoritmo de matching al cliente: el badge del header decía literalmente `"matching Jaccard + trust"` y el subtítulo hacía referencia explícita al "Algoritmo de matching".

## Solución

Se reemplazaron las etiquetas técnicas por copys orientadas al usuario en `apps/web/app/(app)/client/professionals/page.tsx`:

- Subtítulo del `ClientPageHeader`: `"Algoritmo de matching por compatibilidad con tu trabajo"` → `"Encuentra profesionales compatibles con tu trabajo"`.
- Badge de acción: `"matching Jaccard + trust"` → `"Matching SEMSE"`.
- Leyenda de ordenamiento: `"Ordenados por trust score"` → `"Ordenados por confianza"`.
- Etiqueta del puntaje individual: `"trust"` → `"confianza"`.
