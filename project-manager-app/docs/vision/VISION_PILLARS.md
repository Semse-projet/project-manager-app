# Vision Pillars

## Objetivo

Definir los pilares que deben mantenerse aunque cambie la implementacion concreta.

## Pilar 1. Trabajo real, no software aislado

SEMSE existe para coordinar trabajo real entre personas y organizaciones.

Eso implica:

- alcance claro;
- evidencia real;
- pagos reales;
- trazabilidad real;
- consecuencias operativas reales.

## Pilar 2. Confianza basada en comportamiento

La confianza no debe depender solo de:

- marketing;
- reputacion superficial;
- estrellas sin contexto.

Debe depender de:

- cumplimiento;
- evidencia;
- tiempos;
- disputas;
- consistencia operativa;
- historial verificable.

## Pilar 3. Dinero controlado por reglas

El sistema no solo conecta partes.
Tambien controla:

- escrow;
- liberaciones;
- holdbacks;
- refunds;
- reconciliacion;
- auditoria de movimientos.

## Pilar 4. Operacion auditable

Cada accion importante debe poder explicarse.

Eso requiere:

- audit log;
- timeline;
- ownership;
- permisos;
- reglas de transicion;
- criterio de excepcion.

## Pilar 5. Agentes como aceleradores, no humo

La IA debe entrar donde agrega control o velocidad.

Casos utiles:

- planificacion;
- revision de evidencia;
- alertas;
- soporte operativo;
- resumen de disputas;
- deteccion de riesgo.

No debe usarse solo para parecer avanzada.

## Pilar 6. Arquitectura compatible con crecimiento

El sistema debe soportar:

- MVP curado;
- operaciones mas complejas;
- trust layer;
- futura governance.

Sin exigir reescritura total.

## Pilar 7. Gobernanza futura sin contaminar el MVP

Prometeo importa.

Pero:

- no debe destruir foco;
- no debe imponer complejidad prematura;
- no debe mezclarse con el happy path inicial.

Se preserva como norte institucional.

## Pilar 8. Seguridad antes que conveniencia

Si hay conflicto entre:

- facilidad de implementar;
- UX comoda;
- seguridad del recurso;

gana la seguridad del recurso.

## Pilar 9. Verticalizacion inteligente

El sistema no debe arrancar intentando abarcar todo.

Debe empezar en:

- pocos trades;
- flujos auditables;
- tickets controlados;
- disputes resolubles;
- operaciones repetibles.

## Pilar 10. Evolucion institucional

El largo plazo no es solo escalar volumen.

Es evolucionar hacia:

- confianza portable;
- coordinacion distribuida;
- reglas programables;
- identidad soberana;
- economia interna auditable.
