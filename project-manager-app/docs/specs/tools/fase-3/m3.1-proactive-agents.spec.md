# Spec: M3.1 Proactive Rule-Based Agents

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until each proactive agent has code, fixtures and tests.

## Problem Statement

SEMSE should surface operational risk before users manually discover it: budget burn, lien deadlines, scope creep, weather schedule risk, overdue milestones and cash-flow risk.

## Scope

- In scope:
  - Budget Burn Agent.
  - Lien Deadline Agent.
  - Scope Creep Agent.
  - Weather-Schedule Agent.
  - Milestone Overdue Agent.
  - Cash Flow Risk Agent.
- Out of scope:
  - Fully autonomous dispute resolution.
  - Fully autonomous escrow release.

## Agent Contract

```yaml
agent_output:
  signalType: string
  severity: "low | medium | high | critical"
  entityType: string
  entityId: string
  recommendation: string
  confidence: number
  requiresHumanApproval: boolean
```

## Tests Required

- [x] Each proactive agent emits deterministic signal for fixture case.
- [x] Low-confidence NLP signal requires manual review.
- [x] Financial/legal actions are recommendation-only.
