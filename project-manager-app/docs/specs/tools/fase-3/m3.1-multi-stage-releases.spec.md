**Objetivo:** Gestionar releases de escrow en etapas (draws) en lugar de lump-sum.

---

## Overview

Sistema de draws (solicitudes de pago) donde:
1. PRO solicita draw (DRAFT)
2. Lender revisa (PENDING_LENDER)
3. Lender aprueba (APPROVED)
4. Dinero se libera (FUNDED)
5. Retainage se holdea (5-10% hasta project completion)

---

## DrawRequest FSM

```
DRAFT
  ↓ (submit for approval)
PENDING_LENDER
  ├─ (approve) → APPROVED
  │                ↓ (fund)
  │             FUNDED
  └─ (reject) → REJECTED
```

**Estados:**
- **DRAFT:** Creado pero no enviado
- **PENDING_LENDER:** Esperando aprobación del lender
- **APPROVED:** Lender aprobó (dinero autorizado pero no transferido)
- **FUNDED:** Dinero transferido a PRO
- **REJECTED:** Lender rechazó

---

## DrawRequest Data Model

```ts
DrawRequest {
  id: string;
  projectId: string;
  drawNumber: number; // 1, 2, 3, 4
  amount: number; // $ solicitado
  percentage: number; // % del budget
  retainage: number; // $ holdedo (5-10%)
  status: 'DRAFT' | 'PENDING_LENDER' | 'APPROVED' | 'FUNDED' | 'REJECTED';
  
  // Aprobaciones
  submittedBy: string; // PRO user
  submittedAt: Date;
  approvedBy?: string; // Lender user
  approvedAt?: Date;
  rejectionReason?: string;
  
  // Fundeo
  fundedAt?: Date;
  fundingTransactionId?: string;
  
  // Tiempos
  createdAt: Date;
  updatedAt: Date;
}
```

## Flujo Típico (4 Draws)

```
PROJECT BUDGET: $400,000

DRAW 1 (20%): $80,000
  → Retainage: $8,000 (10%)
  → Neto: $72,000
  → Status: 30-40% completo

DRAW 2 (25%): $100,000
  → Retainage: $10,000 (10%)
  → Neto: $90,000
  → Status: 50-60% completo

DRAW 3 (30%): $120,000
  → Retainage: $12,000 (10%)
  → Neto: $108,000
  → Status: 80-85% completo

DRAW 4 (25%): $100,000
  → Retainage: $0 (final draw, retainage released from previous)
  → Neto: $100,000 + $40,000 (retainage) = $140,000
  → Status: 100% completo
```

## Gates & Conditions

### Antes de DRAFT → PENDING_LENDER:
- [ ] Waiver requirements met (liens)
- [ ] Change orders approved
- [ ] Project status OK
- [ ] No outstanding disputes

### Antes de PENDING_LENDER → APPROVED:
- [ ] Lender verifies work % complete
- [ ] Lender verifies expenses
- [ ] No lien claims pending
- [ ] Insurance current

### Antes de APPROVED → FUNDED:
- [ ] Lender authorization confirmed
- [ ] Escrow account verified
- [ ] ACH/wire initiated

---

## Retainage Logic

**Retainage:** 5-10% del draw amount, held by lender until project completion.

```
Draw 1: $80k → Hold $8k, Pay $72k
Draw 2: $100k → Hold $10k, Pay $90k
Draw 3: $120k → Hold $12k, Pay $108k
Draw 4: $100k → Hold $0k (final), Pay $100k + $30k (retainage releases) = $130k

Total Retainage: $40k (10% of $400k)
Released on: Project Completion + Lien Waiver conditions
```

## Endpoints (M3.1-A)

**POST /v1/projects/:projectId/draws**
- Create new draw (DRAFT)
- Input: amount, % of budget

**GET /v1/projects/:projectId/draws**
- List all draws

**POST /v1/projects/:projectId/draws/:drawId/submit**
- Submit for lender approval (DRAFT → PENDING_LENDER)

**POST /v1/projects/:projectId/draws/:drawId/approve** [LENDER]
- Approve draw (PENDING_LENDER → APPROVED)

**POST /v1/projects/:projectId/draws/:drawId/fund**
- Mark as funded (APPROVED → FUNDED)

**GET /v1/projects/:projectId/retainage**
- Get total retainage held

## Integration Points

**With Liens (Fase 2.1):**
- Before funding: Check waiver requirements
- Gate: Cannot release if liens pending

**With Change Orders (Fase 2.2):**
- Before funding: Verify approved changes included
- Impact: Change amount affects draw %

**With Weather (Fase 2.3):**
- Forecast: Adjust draw schedule based on delays
- Retainage release: Delayed if weather impacts completion
