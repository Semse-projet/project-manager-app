# SPEC: Módulo 1.1 — Precios de Materiales en Tiempo Real
**Dominio:** packages/tools + packages/integrations
**Versión:** 1.0
**Fecha:** 2026-05-20
**Estado:** APPROVED
**Bloques cubiertos:** 1.1.A, 1.1.B, 1.1.C, 1.1.D, 1.1.E

---

## Objetivo

Reemplazar los costos de materiales hardcodeados en los 25 motores de trade con datos reales provenientes de APIs públicas gratuitas, almacenados en caché en la base de datos y actualizados cada 24 horas.

**Problema actual:** Los motores usan valores fijos (e.g., lumber $0.85/LF). Con la volatilidad de materiales post-2025, esto genera errores de estimación de 15-30%.

---

## Fuentes de Datos

### BLS PPI API (gratuita, sin key)
- URL: `https://api.bls.gov/publicAPI/v2/timeseries/data/`
- Series relevantes:
  | Serie | Material |
  |---|---|
  | WPU081 | Lumber & wood products |
  | WPU101 | Metals (steel, iron) |
  | WPU102 | Copper & brass |
  | WPU132 | Gypsum / drywall |
  | WPU133 | Concrete products |
  | WPU134 | Asphalt / roofing felts |
  | WPU061 | Industrial chemicals (paint) |
  | WPU135 | Plumbing fixtures |
  | WPU136 | Electrical products |
- Frecuencia: mensual (2 semanas de lag)
- Rate limit: 25 queries/día sin key; 500/día con key BLS gratuita

### EstimationPro API (gratuita, sin registro)
- URL: `https://estimationpro.ai/api/v1`
- Endpoints:
  | Endpoint | Datos |
  |---|---|
  | GET /trades | Lista de 32 trades con IDs |
  | GET /costs?trade={id} | 385 ítems de costo con unidades |
  | GET /index | EPCI cost index nacional |
  | GET /multipliers?zip={zip} | Multiplicadores por metro |
  | GET /trends | Tendencias de volatilidad |
- Frecuencia de actualización: semanal

### FRED API (gratuita, key gratuita)
- URL: `https://api.stlouisfed.org/fred/series/observations`
- Key: registrar en fred.stlouisfed.org (gratis, instantáneo)
- Series: WPU081 (lumber), construcción, permisos, housing starts
- Frecuencia: mensual

## Nuevo Modelo Prisma: MaterialPriceSnapshot

```prisma
model MaterialPriceSnapshot {
  id          String   @id @default(cuid())
  tenantId    String?  // null = global snapshot
  source      String   // "BLS_PPI" | "ESTIMATION_PRO" | "FRED"
  seriesId    String   // e.g., "WPU081"
  materialKey String   // e.g., "lumber" | "steel" | "copper"
  unit        String   // e.g., "per_mbf" | "per_lb" | "per_sqft"
  value       Float    // precio actual
  indexValue  Float?   // índice (base 100)
  changeYoY   Float?   // cambio % año a año
  periodDate  DateTime // fecha del dato
  fetchedAt   DateTime @default(now())
  expiresAt   DateTime // fetchedAt + 24h
  raw         Json?    // respuesta raw de la API

  @@index([materialKey, source])
  @@index([expiresAt])
}
```

### material-engine.ts — Lookup dinámico
```typescript
// Antes:
const LUMBER_COST_PER_LF = 0.85

// Después:
async function getMaterialCost(
  materialKey: string,
  priceOverrides?: Record<string, number>
): Promise<number> {
  if (priceOverrides?.[materialKey]) return priceOverrides[materialKey]
  const snapshot = await pricingCache.getOrFetch(materialKey, 'BLS_PPI')
  return snapshot?.value ?? MATERIAL_FALLBACK_COSTS[materialKey]
}

// Fallback hardcoded (usado si APIs fallan)
const MATERIAL_FALLBACK_COSTS: Record<string, number> = {
  lumber:   0.85,  // per LF
  steel:    1.20,  // per LF
  copper:   4.50,  // per LF
  drywall:  0.55,  // per sqft
  concrete: 145,   // per yd³
  // ... etc
}
```

## Criterios de Aceptación

1. `npm run verify:workspace` pasa sin errores
2. Los 25 motores de trade compilan con el nuevo lookup dinámico
3. Un motor con `useLivePricing=true` y BLS API disponible devuelve precio real
4. Un motor con BLS API caída devuelve precio del fallback (no falla)
5. El worker job refreshea precios cada 24h
6. AuditLog registra cada refresh
