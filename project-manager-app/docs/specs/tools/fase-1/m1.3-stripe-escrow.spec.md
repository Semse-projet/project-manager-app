# Spec: M1.3 Stripe Connect Escrow

## Problem Statement

SEMSE must support real contractor payouts through Stripe Connect while keeping milestone escrow release governed by evidence, approvals, disputes and auditability.

## Scope

- In scope:
  - Stripe Connect account onboarding.
  - Manual payout/transfer flow after milestone approval.
  - Platform fee deduction.
  - Escrow release service integration.
  - Payment transaction recording.
- Out of scope:
  - Plaid ACH verification, still pending in master plan.
  - Autonomous release without human/payment-governance gates.

## Contract

```yaml
stripe_account:
  contractorId: string
  accountId: string
  onboardingStatus: string
escrow_release:
  milestoneId: string
  paymentId: string
  contractorAccountId: string
  platformFeeRate: 0.0075
  releaseMode: "manual_after_approval"
forbidden_behavior:
  - "release without milestone approval"
  - "release while dispute is open"
  - "release without auditable transaction"
```

## Required Behavior

- Use payment-governance readiness before transfer.
- Deduct SEMSE platform fee on payout.
- Record `PaymentTxn` for release.
- Surface Stripe provider errors without losing escrow state.
