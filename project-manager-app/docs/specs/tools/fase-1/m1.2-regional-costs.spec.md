# Spec: M1.2 Regional Cost Adjustment

## Problem Statement

National average estimates are not reliable enough for contractor pricing. SEMSE ProTools must adjust labor and material estimates by region using OEWS labor snapshots, ZIP/state multipliers and contractor overrides.

## Scope

- In scope:
  - BLS OEWS labor rate snapshots.
  - ZIP/state location cost multipliers.
  - `zipCode` and `location` support in ProTools inputs.
  - `applyLocation()` adjustment in cost engine.
  - Contractor manual rate overrides.
- Out of scope:
  - Paid RSMeans/NECA/PHCC calibration; see `m3.3-labor-calibration`.
  - Real-time permit or tax calculations.

## Data Contract

```yaml
location_input:
  zipCode: "optional string"
  location:
    state: "optional"
    materialMultiplier: "number"
    laborMultiplier: "number"
labor_snapshot:
  source: "BLS_OEWS"
  socCode: string
  state: string
  hourlyRate: number
  fetchedAt: string
contractor_override:
  contractorId: string
  trade: string
  hourlyRate: number
  effectiveFrom: string
```

## Required Behavior

- Use contractor override when present and allowed.
- Fall back to OEWS/state multiplier when no contractor override exists.
- Fall back to national/default pricing when external data is stale or unavailable.
- ProTools result must expose whether regional adjustment was applied.
