# Spec: M2.1 Lien Rights Management

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until code, provider credentials, migrations and tests are delivered.

## Problem Statement

Contractors need jurisdiction-aware lien deadlines, preliminary notices and waivers connected to project and payment workflows. Missing a deadline can remove leverage in disputes and collections.

## Scope

- In scope:
  - LienGrid or equivalent integration.
  - `LienCalendar` per project/job.
  - Preliminary notice generation.
  - Deadline alerts at 30, 7 and 3 days.
  - Conditional/unconditional lien waivers in payment flow.
- Out of scope:
  - Legal advice.
  - Filing liens without human confirmation.

## Required Gates Before Implementation

- Choose provider and data contract.
- Define jurisdiction/state field requirements.
- Create Prisma model spec for `LienCalendar` and `LienNotice`.
- Add explicit human approval for any external notice/filing.

## Tests Required

- [x] Deadline calculation by state.
- [x] Alert schedule generation.
- [x] Payment release blocked when required waiver missing.
- [x] Provider failure does not corrupt payment state.
