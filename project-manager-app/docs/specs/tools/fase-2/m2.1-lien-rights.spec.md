# Spec: Lien Rights Management (LienGrid API)

> **Propósito:** Automatizar deadlines de preliminary notice, waivers, y calendarios de liens en 50 estados US.
> 
> **Para quién:** Contratista que necesita protección legal automática · OPS_ADMIN que supervisa compliance · Cliente que necesita certainty en escrow.
> 
> **Privacidad:** `true` — Project address es location-critical, encriptar en reposo.

---

## 1. Qué Resuelve

El contratista debe presentar "preliminary notices" en plazos muy breves (25-35 días en muchos estados) o pierde derechos de lien.

**Problema:**
- 50 estados = 50 plazo diferentes
- Plazo perdido = pérdida de derechos legales
- Abogados cobran $300-500 por notice
- Contratista no sabe ni que existe

**Solución:**
1. Al crear proyecto: capturar dirección y estado
2. Consultar LienGrid API para deadlines automáticos
3. Crear calendario con alertas (30 días, 7 días, 3 días, 1 día)
4. Generar preliminary notices pre-poblados con datos del proyecto
5. Enviar vía correo certificado digital (Lob.com)
6. Capturar firma de waiver en milestones

**Resultado:** Contratista protegido automáticamente. Cero trabajo manual.

## 3. Máquina de Estados — Lien Calendar

**Entidad:** `LienCalendar` (uno por proyecto + estado)

```
                   Proyecto creado
                    + dirección
                         │
                         ▼
   PENDING ◄─────────────────────────────┐
      │                                   │
  LienGrid consulta                       │
  deadlines ley                           │
      │                                   │
      ▼                                   │
   CREATED                              RETRY
      │                                   ▲
  30 días antes                           │
  (alerta)                                │
      │                                   │
      ▼                                   │
  ALERTED_30D ───────── NO NOTICE SENT ──┘
      │
  7 días antes (2ª alerta)
      │
      ▼
  ALERTED_7D
      │
  3 días antes (3ª alerta)
      │
      ▼
  ALERTED_3D
      │
  Notice enviado vía Lob.com
      │
      ▼
  NOTICE_SENT
      │
  ¿Confirmación de entrega?
      │
      ├──→ SÍ ──→ NOTICE_DELIVERED
      │
      └──→ NO ──→ DELIVERY_FAILED (vuelve a RETRY)

[TERMINAL: NOTICE_DELIVERED, NOTICE_FAILED]
[INVARIANTE: un único calendar por proyecto+estado]
```

### Transiciones

| Desde | Hacia | Trigger | Actor | Efecto |
|-------|-------|---------|-------|--------|
| `PENDING` | `CREATED` | LienGrid API responde con deadlines | PLATFORM (scheduler) | Guardar deadlines en BD |
| `CREATED` | `ALERTED_30D` | T-30 días antes deadline | PLATFORM (scheduler) | Push notification + email al PRO |
| `ALERTED_30D` | `ALERTED_7D` | T-7 días antes deadline | PLATFORM (scheduler) | 2ª notificación (urgencia) |
| `ALERTED_7D` | `ALERTED_3D` | T-3 días antes deadline | PLATFORM (scheduler) | 3ª notificación (final) |
| `ALERTED_3D` | `NOTICE_SENT` | PRO acepta envío O sistema auto-envía | PLATFORM | Crear PDF notice + enviar Lob |
| `NOTICE_SENT` | `NOTICE_DELIVERED` | Webhook de Lob: entrega confirmada | PLATFORM (webhook) | Audit log + mark as complete |
| `NOTICE_SENT` | `DELIVERY_FAILED` | Webhook de Lob: bounce/error | PLATFORM (webhook) | Volver a RETRY |
| cualquiera | `RETRY` | Error en integración | PLATFORM | Reintentar (max 3 veces) |

## 4. Contratos de API

### 4.1 Integración con LienGrid

**Endpoint LienGrid:** `POST https://api.liengrid.com/v1/deadlines`

**Request:**
```json
{
  "address": "123 Main St, San Francisco, CA 94102",
  "state": "CA",
  "projectStartDate": "2026-07-01",
  "projectDescription": "Roof replacement",
  "contractAmount": 50000,
  "apiKey": "${LIENGRID_API_KEY}"
}
```

**Response:**
```json
{
  "state": "CA",
  "preliminaryNoticeDeadline": "2026-08-10",
  "waiverDeadline": "2026-09-15",
  "finalNoticeDeadline": "2026-10-20",
  "statusLienDeadline": "2026-12-20",
  "requiresNotary": false,
  "requiresCertifiedMail": true,
  "recipientTypes": ["owner", "general_contractor", "lender"]
}
```

## 5. Integración con Payments FSM

**Gate antes de `RELEASED`:**

```ts
// En PaymentGovernanceService
async authorizeRelease(escrowId: string): Promise<boolean> {
  const escrow = await prisma.paymentEscrow.findUnique({ 
    where: { id: escrowId },
    include: { milestone: { include: { project: true } } }
  });

  // Check: ¿hay liens pendientes para esta release?
  const pendingWaivers = await prisma.lienWaiver.findMany({
    where: {
      lienCalendar: { projectId: escrow.milestone.project.id },
      releaseAmount: escrow.amount,
      status: 'PENDING'
    }
  });

  if (pendingWaivers.length > 0) {
    return false; // BLOCK RELEASE
  }

  // Approve release
  return true;
}
```

**Efecto:** Cliente no puede liberar escrow hasta que PRO firme waiver.

## 6. Integración con Change Orders

**Ante un change order aprobado:**

1. Recalcular lien deadlines (si aumentó contract amount)
2. Crear nuevas alertas si es necesario
3. Notificar PRO de deadline extendido

```ts
async onChangeOrderApproved(changeOrder: ChangeOrder) {
  const { projectId, newContractAmount } = changeOrder;
  const lienCals = await prisma.lienCalendar.findMany({
    where: { projectId }
  });

  for (const cal of lienCals) {
    // Reconsultar LienGrid con nuevo monto
    const updated = await liengrid.getDeadlines({
      contractAmount: newContractAmount,
      state: cal.stateName
    });
    
    // Actualizar si cambió
    await prisma.lienCalendar.update({
      where: { id: cal.id },
      data: updated
    });
  }
}
```

## 7. Casos de Uso

### UC-2.1.1: Crear Proyecto en California

**Actor:** PRO (contratista)

1. Crear proyecto con dirección en California
2. Sistema consulta LienGrid → deadline 2026-08-10
3. Crear LienCalendar con estado CREATED
4. Schedulear alertas para 30d, 7d, 3d, 1d antes

**Resultado:** PRO recibe notificaciones automáticas. Sin hacer nada.

### UC-2.1.2: Alerta de Preliminary Notice (7 días antes)

**Actor:** PLATFORM (scheduler)

1. Cada hora, buscar LienCalendars donde T-7 días = hoy
2. Cambiar a ALERTED_7D
3. Enviar push + email: "Preliminary notice due in 7 days"
4. Incluir link "Send Now" que llama a send-notice endpoint

**Resultado:** PRO puede enviar con 1 click. O ignorar y recibir alerta final en 3 días.

### UC-2.1.3: Enviar Preliminary Notice

**Actor:** PRO o PLATFORM (auto)

1. PRO hace click en "Send Now" o system auto-sends en deadline-1d
2. Sistema genera PDF pre-poblado con:
   - Project address, description, start date
   - Contractor info (desde Profile)
   - Required recipients (from LienGrid response)
3. Llamar Lob.com API para envío certificado
4. Guardar tracking ID de Lob
5. Cambiar estado a NOTICE_SENT

**Resultado:** Notice enviado vía USPS certified mail. Tracking automático.

### UC-2.1.4: Recibir Confirmación de Entrega

**Actor:** PLATFORM (webhook from Lob.com)

1. Lob.com webhook: `letter_delivered` event
2. Actualizar LienCalendar a NOTICE_DELIVERED
3. Emit event: `lien.notice_delivered`
4. Agents pueden reaccionar (ej: Auto-unlock escrow waiver)

**Resultado:** Audit trail completo. System knows notice was delivered.

### UC-2.1.5: Firmar Conditional Waiver (antes de release)

**Actor:** PRO (cuando hace click en Payment Release)

1. System detecta pending waivers
2. Redirigir a página de firma digital
3. PRO firma waiver condicional
4. Guardar firma + timestamp
5. Actualizar LienWaiver a SIGNED
6. Unlock escrow release

**Resultado:** Client puede liberar escrow. PRO protegido legalmente.

## 8. Notas Técnicas

### LienGrid API Cost Model

- **Setup:** $0 (no hay setup fee)
- **Per-notice:** $0.50 - $2.00 (variación por estado)
- **Estimated:** ~$5-20 por proyecto (1-10 notices)

**Implementation:** Stripe-like API key, consumo por API call.

### Lob.com Integration

- **Setup:** $0 (no setup fee)
- **Per-letter:** ~$0.50 (USPS first class + handling)
- **API:** REST, webhooks para delivery tracking
- **Rate limit:** 100 reqs/sec (suficiente)

### Retry Strategy

- **LienGrid API fails:** Reintentar hasta 3 veces con backoff exponencial
- **Lob.com send fails:** Reintentar hasta 5 veces (más crítico)
- **Webhook not received:** Manual resend vía OPS endpoint

### Privacy & Security

- **Address:** Encriptado en reposo (AES-256)
- **API keys:** Nunca enviar al frontend; solo backend
- **Waivers:** Firmados digitalmente (no plain text)
- **Audit log:** Toda transacción de notice/waiver queda registrada

### Testing

- **Mock LienGrid:** Fixture de respuestas por estado
- **Mock Lob.com:** Fixture de tracking IDs
- **Unit tests:** Máquina de estados, transitions
- **Integration tests:** Full flow creation → delivery

No necesita datos reales; todo simulable.
