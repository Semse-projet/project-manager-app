# Spec: Anti-Dispute Documentation System

> **Propósito:** Capturar evidencia timestamped, daily logs, change order trails y extended metrics para defensa legal automática.
> 
> **Para quién:** Contratista necesita defensa · OPS_ADMIN que previene disputas · Abogado que necesita documentación.
> 
> **Privacidad:** `true` — Fotos con GPS, registros diarios, cambios son location/behavior-critical.

---

## 1. Qué Resuelve

Cuando una disputa llega, el abogado del contratista necesita:
- ✅ Evidencia de cada fase del trabajo
- ✅ Timestamps precisos (no "sometime in July")
- ✅ GPS de ubicación (no "I think it was on Main St")
- ✅ Registro diario de actividad (crew, tareas, clima, delays)
- ✅ Trail de cambios (change order → firma → ejecución)
- ✅ Métricas de calidad específicas del trade

**Hoy:** Contratista corre alrededor juntando fotos, recordando fechas, buscando emails de cambios.

**Futuro:** SEMSE auto-captura todo. Abogado recibe bundle PDF listo para corte.

---

## 2. Actores y Permisos

| Actor | Rol | Puede hacer | No puede hacer |
|-------|-----|-------------|----------------|
| Crew (campo) | `CREW` | Subir fotos · Crear daily log | Modificar fotos antiguas · Borrar registros |
| Superintendente | `SUPER` | Validar daily log · Firmar cambios | Modificar después de cierre de día |
| PRO (contratista) | `PRO` | Ver documentación · Exportar bundle | Borrar fotos del proyecto |
| Abogado | `COUNSEL` | Ver todo · Exportar para corte | Modificar datos |
| OPS Admin | `OPS_ADMIN` | Audit · Resend notices · Investigate disputes | Cambiar datos históricos |

**Permisos:**
- `evidence:photos:upload` — subir fotos (crew)
- `evidence:daily:create` — crear daily log (super)
- `evidence:export:bundle` — exportar PDF (PRO + COUNSEL)
- `admin:evidence:audit` — ver audit trail (OPS)

## 3. FSM — Foto con Timestamp + GPS

**Entidad:** `EvidencePhoto` (extendida)

```
UPLOAD
   │
   ├─→ Validar EXIF
   │   ├─→ Timestamp OK
   │   └─→ GPS present
   │
   ▼
VALIDATED
   │
   ├─→ Crew puede agregar descripción
   │
   ▼
CATEGORIZED
   │
   ├─→ Foto enlazada a milestone
   │   o change order
   │
   ▼
LINKED
   │
   ├─→ Incluida en daily log
   │
   ▼
FINALIZED

[TERMINAL: FINALIZED]
[INVARIANTE: nunca modificar timestamp o GPS después de VALIDATED]
```

## 4. FSM — Daily Log

**Entidad:** `DailyLog`

```
CREATED (superintendente inicia log del día)
   │
   ├─→ Agregar crew names
   ├─→ Agregar tareas completadas/atrasadas
   ├─→ Registrar observaciones climáticas
   ├─→ Enumerár deliverables
   │
   ▼
REVIEW (final del día, antes de las 6pm)
   │
   ├─→ Supervisor valida
   │   ├─→ Fotos OK
   │   ├─→ Tareas coherentes
   │   ├─→ Notas climáticas consistentes
   │
   ▼
SIGNED (super firma digitalmente)
   │
   ├─→ Timestamp de firma = cierre del día
   ├─→ Inmutable después de firma
   │
   ▼
SUBMITTED

[TERMINAL: SUBMITTED]
[INVARIANTE: no modificar después de firma]
```

```
   │
   ▼
IN_PROGRESS

[TERMINAL: IN_PROGRESS, REJECTED]
[INVARIANTE: no ejecutar sin firmas + fotos]
```

## 6. Contratos de API

### 6.1 Upload de Foto con EXIF Validation

**POST /v1/projects/:projectId/evidence/photos**

```bash
curl -X POST http://localhost:4000/v1/projects/proj_abc/evidence/photos \
  -H "Authorization: Bearer ${TOKEN}" \
  -F "file=@/path/to/photo.jpg" \
  -F "category=before_phase" \
  -F "description=Roof prep, West side"
```

**Backend valida:**
1. Archivo es JPEG/PNG válido
2. EXIF contiene DateTimeOriginal
3. EXIF contiene GPS (latitude, longitude, altitude)

Response:
```json
{
  "id": "photo_123",
  "projectId": "proj_abc",
  "uploadedBy": "crew_user_id",
  "uploadedAt": "2026-06-21T14:32:00Z",
  "exif": {
    "timestamp": "2026-06-21T14:30:00Z",
    "latitude": 37.7749,
    "longitude": -122.4194,
    "altitude": 15,
    "cameraModel": "iPhone 13"
  },
  "category": "before_phase",
  "description": "Roof prep, West side",
  "status": "VALIDATED",
  "url": "https://cdn.semse.app/proj_abc/photo_123.jpg"
}
```

**Error si EXIF incompleto:**
```json
{
  "error": "EXIF_INVALID",
  "details": "Photo missing GPS coordinates",
  "status": 400
}
```

## 7. Extended Metrics (Revisado)

Para los 20 trades sin confidence score actualmente:

**Trades que necesitan extended metrics:**

1. Carpentry (framing, trim, door/window installation)
2. HVAC (ductwork, equipment installation, commissioning)
3. Plumbing (rough-in, fixtures, inspection passes)
4. Electrical (rough-in, panel, trim, inspection passes)
5. Drywall/Interior finishes
6. Masonry/Concrete finishing
7. Insulation/Air sealing
8. Painting/Staining
9. Flooring (installation, finishing)
10. Siding/Exterior finishes
11. Soffit/Fascia/Gutters
12. Doors/Windows (full assembly)
13. Stairs/Railings
14. Cabinetry
15. Countertops
16. Tile/Backsplash
17. Plumbing fixtures (final)
18. Hardware installation
19. Caulk/Sealant
20. Final cleanup/restoration

**Métrica base:** % de trabajo completado vs. planificado + % de defects found in QA

**Métrica extendida por trade:**
- Carpentry: % frames square, % doors/windows level, % trim gaps
- HVAC: % ducts sealed, % equipment operational, % thermostat calibrated
- Plumbing: % rough-in passes inspection, % pressure test OK, % fixtures installed to spec
- Electrical: % circuits protected, % grounding OK, % inspection passes
- Drywall: % taping smooth, % finish level, % paint coverage
- Masonry: % grout consistent, % joints tooled, % sealing applied
- etc.

**Captura:** Crew foto + descripción, sistema extrae metrics de daily log.

## 8. Integración con Milestones FSM

**Gate antes de `COMPLETE`:**

```ts
async onMilestoneComplete(milestone: Milestone) {
  // 1. ¿Existen daily logs para este milestone?
  const logs = await prisma.dailyLog.findMany({
    where: {
      projectId: milestone.projectId,
      date: { gte: milestone.startDate, lte: milestone.targetDate }
    }
  });
  
  if (logs.length === 0) {
    throw new Error('No daily logs found — cannot complete milestone');
  }

  // 2. ¿Existen fotos?
  const photos = await prisma.evidencePhoto.findMany({
    where: {
      projectId: milestone.projectId,
      uploadedAt: { gte: milestone.startDate, lte: milestone.targetDate },
      status: 'FINALIZED'
    }
  });

  if (photos.length === 0) {
    throw new Error('No photos found — cannot complete milestone');
  }

  // 3. ¿Extended metrics OK?
  // (verificar por trade)
  
  // OK → puede proceder a COMPLETE
}
```

## 9. Casos de Uso

### UC-2.2.1: Crew Sube Foto en Campo

**Actor:** Crew con smartphone

1. Crew abre SEMSE app en campo
2. Toma foto con GPS automático
3. Selecciona categoría (before/during/after)
4. Escribe descripción breve
5. Click "Upload"

**Sistema:**
1. Validar EXIF (timestamp + GPS)
2. Subir a CDN
3. Marcar como VALIDATED
4. Notificar superintendente

**Resultado:** Foto timestamped + GPS capturada automáticamente.

### UC-2.2.2: Superintendente Crea Daily Log (final del día)

**Actor:** Superintendente en campo

1. 5:45pm — Abrir app SEMSE
2. "Create Daily Log" para hoy
3. Agregar crew names (prepoblado de checkin)
4. Agregar tareas completadas/atrasadas
5. Nota clima: "Rain 2-4pm, halted roofing work"
6. Sistema muestra fotos del día → super confirma
7. Click "Sign" → firma digital capturada
8. Log SUBMITTED

**Resultado:** Registro diario firmado, inmutable.

### UC-2.2.3: Change Order Trail

**Actor:** GC/Superintendent

1. Detect necesidad de cambio (scope, tiempo, costo)
2. Proponer en SEMSE: descripción + delta costo
3. Sistema genera URL de firma
4. Email a cliente para firmar
5. Cliente firma (firma digital)
6. Sistema verifica ambas partes OK
7. "Ready to Execute" — crew puede proceder

**Resultado:** Change order con trail de decisiones y firmas.

### UC-2.2.4: Export Bundle para Abogado

**Actor:** PRO cuando disputa se abre

1. Login SEMSE
2. Proyecto → "Export for Legal"
3. Click "Generate PDF Bundle"
4. Sistema produce PDF con:
   - Todas las fotos (cronológicamente)
   - Todos los daily logs (del primero al último)
   - Todos los change orders (con firmas)
   - Lien calendar (deadlines, notices)
   - Extended metrics por trade
5. Email PDF a PRO (y abogado si registrado)

**Resultado:** Abogado tiene 90% de documentación que necesita.

## 10. Notas Técnicas

### EXIF Validation Library

```ts
import ExifParser from 'exif-parser';

async function validatePhotoEXIF(buffer: Buffer) {
  const parser = ExifParser.create(buffer);
  const result = parser.parse();
  
  const { DateTimeOriginal, GPSLatitude, GPSLongitude } = result.tags;
  
  if (!DateTimeOriginal) throw new Error('Missing timestamp');
  if (!GPSLatitude || !GPSLongitude) throw new Error('Missing GPS');
  
  return {
    timestamp: new Date(DateTimeOriginal),
    latitude: GPSLatitude,
    longitude: GPSLongitude
  };
}
```

### Daily Log Signature

- SignaturePad.js para captura de firma en web/mobile
- Guardar como base64 PNG
- Encriptar en BD (AES-256)
- Verificar con timestamp

### PDF Generation

- Use `pdfkit` or `puppeteer` para PDF generation
- Template HTML con todos los datos
- Include images inline (base64 encoded)
- Agregar watermark: "Generated by SEMSE, [date]"

### Storage

- Photos: CDN (S3 o similar)
- Daily logs: Base de datos
- PDFs: Generate on-demand (no almacenar, ahorra espacio)

### Privacy & Security

- GPS data: Encriptado en reposo
- Fotos: Encriptadas en tránsito (HTTPS)
- Access control: Solo PRO, crew, counsel pueden ver
- Audit log: Todo acceso registrado
