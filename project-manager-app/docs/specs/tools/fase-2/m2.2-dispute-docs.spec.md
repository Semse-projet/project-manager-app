# Spec: M2.2 Anti-Dispute Documentation

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until code, privacy policy updates, migrations and tests are delivered.

## Problem Statement

Evidence, daily logs, GPS/timestamp metadata and signed change-order trails must be structured so SEMSE can defend milestone approvals, escrow releases and dispute outcomes.

## Scope

- In scope:
  - GPS/timestamp requirements for evidence upload.
  - EXIF/geolocation validation policy.
  - Daily project logs.
  - Signed change-order trail before execution.
  - Evidence export bundle.
- Out of scope:
  - Court filing automation.
  - Legal conclusions from AI.

## Required Gates Before Implementation

- Define privacy policy for GPS metadata.
- Define retention policy for evidence export bundles.
- Add explicit fallback when GPS/EXIF is unavailable.
- Ensure user-facing consent copy exists.

## Tests Required

- [x] Evidence upload with GPS/timestamp.
- [x] Evidence upload without required metadata triggers review/block.
- [x] Export bundle includes evidence, logs and change orders.
- [x] Change order cannot proceed when required signature missing.
