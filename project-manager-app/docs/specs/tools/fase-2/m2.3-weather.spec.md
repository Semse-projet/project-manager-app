# Spec: M2.3 Weather-Aware Project Risk

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until provider integration, scheduling behavior and tests are delivered.

## Problem Statement

Weather can delay construction tasks and create valid change-order or schedule-risk evidence. SEMSE needs project-level weather alerts connected to field logs and BuildOps.

## Scope

- In scope:
  - Tomorrow.io or equivalent provider.
  - Weather alerts by project location.
  - Trade-aware thresholds for rain, wind, freeze and hail.
  - Automatic field log entry when weather affects work.
  - Weather-triggered change-order recommendation.
- Out of scope:
  - Insurance claim automation.
  - Weather-based autonomous schedule changes without confirmation.

## Tests Required

- [x] Provider response mapping.
- [x] Trade-specific alert thresholds.
- [x] Weather alert creates field log.
- [x] Weather alert can recommend, but not auto-approve, a change order.
