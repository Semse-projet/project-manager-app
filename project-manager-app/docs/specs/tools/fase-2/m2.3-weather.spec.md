# Spec: Weather Integration & Alerts (Tomorrow.io)

> **Propósito:** Capturar datos climáticos en tiempo real por proyecto, automatizar alertas de impacto laboral, documentar delays por clima.
> 
> **Para quién:** Superintendente que debe pausar trabajo · PRO que necesita change order justificado · Cliente que entiende delays por clima.
> 
> **Privacidad:** `false` — Datos de clima son públicos, solo ubicación es sensible.

---

## 1. Qué Resuelve

Clima impacta directamente trabajos de construcción. Problemas hoy:

1. **No hay alertas:** Supervisor descubre lluvia cuando crew llega al sitio
2. **No hay documentación:** "Work halted due to weather" sin evidencia = cliente cree es negligencia
3. **No hay change order justificado:** Delays por helada/lluvia no documentados = cliente no paga
4. **No hay trade-awareness:** Igual alerta para roofing (crítico con lluvia) vs. interior pintura (no afectada)

**Solución:**

1. Tomorrow.io API da alertas 24h antes de evento crítico
2. Sistema automaticamente:
   - Notifica superintendente
   - Flag tareas exteriores afectadas
   - Log automático: "work halted due to [weather]"
   - Genera base para change order
3. Sistema entiente trades:
   - Roofing/concrete: HALT todo
   - Siding/painting: HALT exterior
   - Interior: PROCEED
   - Electrical: Partial halt (no outdoor work)

**Resultado:** Delays por clima documentados automáticamente. Sin litigio.

## 3. Data Model — Weather Event

**Entidad:** `WeatherAlert`

```json
{
  "id": "alert_123",
  "projectId": "proj_abc",
  "eventType": "RAIN",
  "severity": "HIGH",
  "startTime": "2026-06-22T14:00:00Z",
  "endTime": "2026-06-22T18:00:00Z",
  "probability": 0.95,
  "maxIntensity": "Heavy rain (1-2 in/hr)",
  "affectedTrades": ["roofing", "concrete", "siding"],
  "notCriticalFor": ["interior_paint", "drywall", "electrical_indoor"],
  "source": "tomorrow.io",
  "alertedAt": "2026-06-21T14:00:00Z",
  "notificationSent": true,
  "autoHaltCreated": false,
  "status": "ACTIVE"
}
```

**Estados:**
- `FORECAST` — alerta futura (>24h)
- `IMMINENT` — alerta próxima (0-24h)
- `ACTIVE` — evento ocurriendo ahora
- `COMPLETED` — evento pasó

## 4. FSM — Weather Alert Workflow

```
FORECAST (>24h antes)
   │
   ├─→ Sistema calcula impacto
   │   ├─→ Qué trades afectados
   │   ├─→ Qué milestones en riesgo
   │   ├─→ Qué tareas deben pausar
   │
   ▼
IMMINENT (0-24h antes)
   │
   ├─→ PUSH NOTIFICATION a superintendente
   │   "Rain expected 2-6pm tomorrow — roofing/concrete affected"
   │   [Button: View Details] [Button: Prepare/Halt]
   │
   ▼
ACTIVE (evento ocurriendo)
   │
   ├─→ Si SUPER confirma → Auto-create halt log
   │   "Work halted due to RAIN, 2-6pm"
   │   Affected tasks paused
   │   Daily log note: auto-added
   │
   ▼
COMPLETED (evento pasó)
   │
   ├─→ SUPER resumes work
   │   ├─→ Verify site conditions
   │   ├─→ Resume paused tasks
   │
   └─→ Sistema genera:
       ├─→ Weather impact summary
       ├─→ Suggested change order scope
       ├─→ Delay calculation

[TERMINAL: COMPLETED]
```

## 8. Integration with Daily Log

**Auto-populate weather observation:**

When superintendente creates daily log and there was a weather event:

```ts
async onDailyLogCreate(log: DailyLog) {
  const alerts = await prisma.weatherAlert.findMany({
    where: {
      projectId: log.projectId,
      startTime: { gte: startOfDay(log.date) },
      status: { in: ['ACTIVE', 'COMPLETED'] }
    }
  });

  if (alerts.length > 0) {
    const alertSummary = alerts
      .map(a => `${a.eventType} (${a.severity})`)
      .join(', ');
    
    log.weatherObservation = `${alertSummary} — work halted`;
  }
}
```

## 9. Integration with Milestone FSM

**Gate before scheduling:**

```ts
async canScheduleMilestone(milestone: Milestone, targetDate: DateTime) {
  // Buscar alertas críticas en la fecha
  const criticalAlerts = await prisma.weatherAlert.findMany({
    where: {
      projectId: milestone.projectId,
      startTime: { gte: targetDate, lte: addDays(targetDate, 1) },
      affectedTrades: { hasSome: milestone.affectedTrades },
      severity: 'CRITICAL'
    }
  });

  if (criticalAlerts.length > 0) {
    return false; // Warn user: bad date
  }

  return true;
}
```

## 10. Casos de Uso

### UC-2.3.1: Alerta de Lluvia Inminente

**Actor:** PLATFORM (scheduler)

1. Cada hora, consultar Tomorrow.io para proyectos activos
2. Detectar evento RAIN con prob >80% en próximas 24h
3. Calcular severidad (intensity × trade impact)
4. Si CRITICAL para algún trade:
   - Cambiar alert status a IMMINENT
   - Push notification: "Heavy rain expected 2-6pm tomorrow — roofing/concrete affected"
   - Email a superintendente con link to "Prepare" / "Acknowledge"

**Resultado:** Superintendente con 24h de aviso.

### UC-2.3.2: Superintendente Confirma Halt por Clima

**Actor:** Superintendent

1. Recibe alert: "Rain starting now, roofing should halt"
2. Click "Confirm Halt"
3. Sistema:
   - Crea daily log entry automáticamente
   - Pausa tareas outdoor de roofing
   - Genera halt log con timestamp

**Resultado:** Sistema sabe que trabajo fue halted por clima, no por negligencia.

### UC-2.3.3: Generar Change Order para Delay

**Actor:** Superintendente o PRO

1. Halted work: 1 día de rain
2. Tareas ahora scheduled 1 día después
3. Click "Request Change Order"
4. Sistema genera CO draft:
   - Description: "Delay due to weather (6/21, rain)"
   - Cost delta: $0 (solo delay, no extra cost)
   - Attached: weather alert data + daily log note
5. Cliente revisa y aprueba
6. Milestone re-scheduled

**Resultado:** Change order justificado automáticamente.

### UC-2.3.4: Trade-Specific Alert

**Actor:** Electrical work, wind alert

1. Wind forecast: 25 mph gusts
2. System checks: Electrical + wind impact = PARTIAL halt (outdoor work only)
3. Notificación: "Wind alert — outdoor electrical work should halt. Interior OK."
4. Superintendent puede:
   - Halt outdoor only (duct-bank pulling, etc)
   - Continue interior (panel installation in garage)

**Resultado:** Trabajo continúa donde es seguro.

## 11. Notas Técnicas

### Polling Strategy

- Poll Tomorrow.io API cada hora (2-3 calls/día)
- Cache resultado por 55 minutos
- Cuando alerta cambia status → broadcast update vía SSE

### Alert Thresholds (por severidad)

```ts
const WEATHER_THRESHOLDS = {
  RAIN: {
    SAFE: { intensity: '<0.1 in/hr', probability: '<50%' },
    ALERT: { intensity: '0.1-0.5 in/hr', probability: '50-80%' },
    CRITICAL: { intensity: '>0.5 in/hr', probability: '>80%' }
  },
  FROST: {
    SAFE: { temp: '>40°F' },
    ALERT: { temp: '32-40°F', duration: '<2h' },
    CRITICAL: { temp: '<32°F', duration: '>4h' }
  },
  WIND: {
    SAFE: { speed: '<10 mph' },
    ALERT: { speed: '10-20 mph' },
    CRITICAL: { speed: '>20 mph' }
  },
  HAIL: {
    SAFE: 'No hail',
    ALERT: 'Hail <0.25 in',
    CRITICAL: 'Hail ≥0.25 in'
  }
};
```

## Apéndice: Matriz de Impacto Detallada

### Roofing

- **Rain >0.1 in/hr:** CRITICAL (water ingress, safety)
- **Frost <32°F:** CRITICAL (material brittleness)
- **Hail:** CRITICAL (damage to completed roof)
- **Wind >20 mph:** CRITICAL (safety, fall risk)

### Concrete

- **Rain <4h after pour:** CRITICAL (curing disrupted)
- **Frost:** CRITICAL (freeze-thaw cycles)
- **Direct sun >95°F:** PARTIAL (accelerated curing)

### Electrical (outdoor/rough-in)

- **Rain:** PARTIAL (only outdoor, not panel install)
- **Wind >20 mph:** PARTIAL (outdoor work)
- **Interior electrical:** SAFE (climate independent)

### Interior Paint

- **Rain:** SAFE (interior protected)
- **Humidity >85%:** ALERT (drying slow)
- **Temp <50°F:** ALERT (paint doesn't adhere)
