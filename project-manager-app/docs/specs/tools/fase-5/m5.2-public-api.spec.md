# Spec: M5.2 Public API and Partner Ecosystem

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until API keys, public versioning, webhook signing and tests are delivered.

## Problem Statement

Partners and larger contractors may need API access to SEMSE estimation engines, webhooks and white-label capabilities. Public access must be rate-limited, authenticated and auditable.

## Scope

- In scope:
  - Public API for ProTools estimate engines.
  - API key issuance and rotation.
  - Rate limiting by tenant/partner/tier.
  - Webhooks with HMAC-256 validation.
  - White-label tenant theming contract.
- Out of scope:
  - Anonymous unauthenticated API access.
  - Exposing internal prompts, provider keys or raw tenant data.

## API Contract

```yaml
partner_api_key:
  tenantId: string
  keyId: string
  scopes: string[]
  rateLimitPerMinute: number
  status: "active | revoked"
webhook_subscription:
  url: string
  secretRef: string
  events: string[]
  hmac: "sha256"
white_label:
  tenantId: string
  themeId: string
  customDomain: string | null
```

## Required Gates Before Implementation

- Define partner scopes.
- Define public API versioning policy.
- Define webhook retry and dead-letter behavior.
- Define white-label theming security boundaries.

## Tests Required

- [x] API key required for public endpoints.
- [x] Rate limit enforced.
- [x] HMAC signature validation for webhook delivery.
- [x] Revoked key cannot access engines.
