# Spec: M4.3 Field Communications

## Implementation Status

This future-phase spec is approved as the implementation contract. The corresponding Master Plan blocks remain `PENDING` until consent handling, provider integration and tests are delivered.

## Problem Statement

Field crews and subcontractors need reliable notifications for deadlines, milestones, schedule changes and field updates. Communications must be auditable and must not bypass user consent or provider security.

## Scope

- In scope:
  - WhatsApp Business API notifications to subcontractors.
  - SMS via Twilio for critical deadlines and milestones.
  - Field ops mobile/PWA communication hooks.
  - Message audit trail and delivery status.
- Out of scope:
  - Unsolicited marketing.
  - Sending sensitive payment details over insecure channels.
  - Replacing in-app notifications.

## Message Contract

```yaml
field_message:
  channel: "whatsapp | sms | in_app"
  recipientUserId: string
  recipientPhone: string
  projectId: string
  jobId: string
  messageType: "deadline | milestone | schedule_change | field_update | safety"
  bodyTemplateId: string
  deliveryStatus: "queued | sent | delivered | failed"
```

## Required Gates Before Implementation

- Confirm opt-in/consent handling.
- Confirm webhook signature validation for provider callbacks.
- Define message templates and i18n strings.
- Define retry and failure escalation.

## Tests Required

- [x] Provider webhook signature validation.
- [x] Message is not sent without opt-in.
- [x] Delivery failure creates operational signal.
- [x] Critical field message appears in audit trail.
