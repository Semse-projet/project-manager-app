# Spec — Verticals Shell

## Objetivo

Crear `/admin/verticals` como entrada a verticales de negocio sin duplicar la app.

## Verticales iniciales

1. Construction
2. Property Turnovers
3. Cleaning
4. Agro / FarmOps
5. Maintenance

## UI requerida

Cada vertical debe mostrar:

- nombre;
- descripción;
- estado;
- número mock de proyectos o plantillas;
- CTA para abrir o configurar.

## Regla técnica

No crear vertical engine real todavía. Solo shell visual y navegación segura.

## Criterios de aceptación

- `/admin/verticals` carga.
- Cards responsive.
- No rompe rutas actuales.
- No requiere backend.
