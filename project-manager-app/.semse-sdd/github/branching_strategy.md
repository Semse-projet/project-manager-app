# GitHub Branching Strategy

## Branch naming

```txt
sdd/admin-modular-navigation
sdd/tool-hub-context-bridge
sdd/workops-flow-shell
sdd/verticals-hub
fix/railway-web-build
fix/typecheck-admin-nav
```

## Commit strategy

```txt
commit 1: add SDD spec and plan
commit 2: add central navigation config
commit 3: add module hub route
commit 4: update Mission Control shell
commit 5: validation fixes
```

## PR size

Prefer small PRs. A PR should be reviewable in under 20 minutes.
